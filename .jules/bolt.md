## 2024-05-17 - Redundant function calls and loop boundary checks
**Learning:** In the `network_manager.py` file within `modules/emulator/src/network_manager`, we were recalculating the `convert_dpid_key()` function for the same variable up to three times during a single pass of a loop over network links, while performing repeated dictionary queries. In `routing.py` within `modules/emulator/src/routing/`, the loop evaluating `dpids` evaluated redundant conditions (`if i + 1 > len(dpids) - 1` and `elif i - 1 < 0`) inside the loop, rather than limiting the iteration bounds of the `range()`.
**Action:** When working with Python loops, prioritize optimizing out unnecessary conditional bounds checking by limiting `range()` limits where possible. When repeatedly using the result of a function on the same input within a loop (like dictionary keys), assign the output to a local variable to prevent duplicate execution overhead and redundant lookups.
